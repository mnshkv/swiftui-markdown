# Markdown Renderer (`Marked`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `Marked` module that turns a Markdown string into the engine's `TextDocument` under a single `MarkdownStyle`, exposed as a one-line SwiftUI `MarkdownView`.

**Architecture:** Three layers — a pure `MarkdownRenderer` core that maps `MarkdownAST` → `MarkdownTextEngine.TextDocument` (fully TDD-tested), a `MarkdownStyle` value config, and a thin SwiftUI `MarkdownView` shell that parses, watches `colorScheme`/`openURL`, and hosts the engine's view. Spec: `docs/superpowers/specs/2026-06-23-renderer-design.md`.

**Tech Stack:** Swift 6.2, Swift Testing, SwiftPM. Depends on `MarkdownAST` (pure parser) + `MarkdownTextEngine` (CoreText engine, Apple-only).

## Global Constraints

- Swift 6.2; Swift Testing (`@Test`/`@Suite`/`#expect`). NO third-party dependencies.
- New module `Marked` depends on `MarkdownAST` + `MarkdownTextEngine`; `@_exported import` both. Apple-only (iOS 26+/macOS 14+).
- The core (`MarkdownRenderer`, `InlineMapper`, `BlockMapper`, `MarkdownStyle`) imports ONLY the AST models, engine models, and CoreGraphics — NO SwiftUI/UIKit/AppKit. SwiftUI lives only in `MarkdownView.swift` behind `#if canImport(SwiftUI)`.
- `render` and `MarkdownParser.parse` are TOTAL — never throw, never crash on any input.
- TDD: every behavior driven by a failing test first. One commit per task.
- Colors are `CGColor`; the active `Palette` is chosen by `colorScheme` and baked into every `TextStyle`.
- Trailing newline on every new file.

## Reference — types this plan builds on

**AST (input):** `MarkdownDocument{blocks:[MarkdownBlock], footnotes:[FootnoteDefinition]}`; `MarkdownBlock`: `.heading(level:Int,content:[MarkdownInline])`, `.paragraph(content:)`, `.blockQuote(blocks:)`, `.list(MarkdownList)`, `.codeBlock(language:String?,code:String)`, `.thematicBreak`, `.table(MarkdownTable)`, `.definitionList([MarkdownDefinition])`; `MarkdownInline`: `.text(String)`, `.emphasis([..])`, `.strong([..])`, `.strikethrough([..])`, `.code(String)`, `.link(destination:String,title:String?,content:[..])`, `.image(source:String,title:String?,alt:String)`, `.autolink(url:String)`, `.footnoteReference(id:String)`, `.softBreak`, `.hardBreak`; `MarkdownList{kind:.bullet|.ordered(start:Int), isTight:Bool, items:[MarkdownListItem{blocks:[MarkdownBlock], task:TaskState?}]}`; `TaskState{.checked,.unchecked}`; `MarkdownTable{alignments:[.none|.left|.center|.right], header:[[MarkdownInline]], rows:[[[MarkdownInline]]]}`; `MarkdownDefinition{term:[MarkdownInline], details:[[MarkdownBlock]]}`; `FootnoteDefinition{id:String, blocks:[MarkdownBlock]}`; `MarkdownParser.parse(_ String) -> MarkdownDocument`.

**Engine (output):** `TextDocument{blocks:[Block]}`; `Block`: `.paragraph(Paragraph{runs:[InlineRun], style:ParagraphStyle})`, `.list(List{marker:ListMarkerStyle, isTight:Bool, items:[TextDocument]})`, `.quote(TextDocument)`, `.table(Table{alignments:[TextAlignment], header:[[InlineRun]], rows:[[[InlineRun]]], cellStyle:TextStyle})`, `.codeBlock(CodeBlock{lines:[String], language:String?, style:TextStyle})`, `.image(ImageAttachment)`, `.thematicBreak(RuleStyle)`; `InlineRun`: `.text(String,TextStyle)`, `.link(runs:[InlineRun], payload:LinkPayload)`, `.inlineImage(ImageAttachment)`, `.lineBreak(hard:Bool)`; `TextStyle{fontSize:CGFloat, isBold,isItalic,isStrikethrough,isMonospace:Bool, color:CGColor}` (init: `TextStyle(fontSize:isBold:isItalic:isStrikethrough:isMonospace:color:)`, all-but-fontSize/color defaulted); `ParagraphStyle{alignment:TextAlignment, lineSpacing,spacingBefore,spacingAfter,leadingIndent:CGFloat}` (all defaulted, `.body`); `ListMarkerStyle{.bullet|.ordered(start:Int)}`; `TextAlignment{.leading,.center,.trailing,.justified}`; `RuleStyle{thickness:CGFloat, color:CGColor}`; `LinkPayload(_ token:String)`; `ImageAttachment{source:String, intrinsicSize:CGSize, alt:String}`; `MarkdownTextView` (SwiftUI), `protocol ImageProvider: Sendable`.

## Wave 2 — Inline mapping

### Task 2.1: `InlineMapper.map` (all inline nodes)

**Files:** Create `Sources/Marked/InlineMapper.swift`; Test `Tests/MarkedTests/InlineMapperTests.swift`

**Consumes:** `StyleContext`, `[MarkdownInline]`, `footnotes: [String: Int]` (id→number).
**Produces:** `enum InlineMapper { static func map(_ nodes: [MarkdownInline], base: TextStyle, ctx: StyleContext, footnotes: [String: Int]) -> [InlineRun] }`.

Implementation (recursive, copy-and-modify; merge handled in Task 2.2):
```swift
enum InlineMapper {
    static func map(_ nodes: [MarkdownInline], base: TextStyle, ctx: StyleContext, footnotes: [String: Int]) -> [InlineRun] {
        var runs: [InlineRun] = []
        for node in nodes {
            switch node {
            case .text(let s): runs.append(.text(s, base))
            case .emphasis(let c):       var st = base; st.isItalic = true;        runs += map(c, base: st, ctx: ctx, footnotes: footnotes)
            case .strong(let c):         var st = base; st.isBold = true;          runs += map(c, base: st, ctx: ctx, footnotes: footnotes)
            case .strikethrough(let c):  var st = base; st.isStrikethrough = true; runs += map(c, base: st, ctx: ctx, footnotes: footnotes)
            case .code(let s):
                var st = base; st.isMonospace = true; st.color = ctx.palette.code; st.fontSize = ctx.style.codeFontSize
                runs.append(.text(s, st))
            case .link(let dest, _, let c):
                runs.append(.link(runs: map(c, base: ctx.linkColored(base), ctx: ctx, footnotes: footnotes), payload: LinkPayload(dest)))
            case .image(let src, _, let alt):
                runs.append(.inlineImage(ImageAttachment(source: src, intrinsicSize: ctx.style.inlineImageSize, alt: alt)))
            case .autolink(let url):
                runs.append(.link(runs: [.text(url, ctx.linkColored(base))], payload: LinkPayload(url)))
            case .footnoteReference(let id):
                let n = footnotes[id] ?? 0
                runs.append(.link(runs: [.text("[\(n)]", ctx.linkColored(ctx.footnote))], payload: LinkPayload("footnote:\(id)")))
            case .softBreak: runs.append(.text(" ", base))
            case .hardBreak: runs.append(.lineBreak(hard: true))
            }
        }
        return merge(runs)   // Task 2.2; in 2.1 `merge` is the identity `{ $0 }`
    }
}
```

  - `.strong([.emphasis([.text("a")])])` → bold AND italic.

  - `.image(source:"s",title:nil,alt:"a")` → `.inlineImage(source=="s", alt=="a", intrinsicSize == 18×18)`.
  - `.footnoteReference("fn")` with `footnotes ["fn":3]` → `.link` text `"[3]"`, payload `"footnote:fn"`.
  - `.softBreak` → `.text(" ", _)`; `.hardBreak` → `.lineBreak(hard: true)`.

## Notes for the executor

- The core mappers (`InlineMapper`, `BlockMapper`, `MarkdownRenderer`, `StyleContext`, `MarkdownStyle`) MUST NOT import SwiftUI/UIKit/AppKit — only `MarkdownAST`, `MarkdownTextEngine`, `CoreGraphics`, `Foundation`. Only `MarkdownView.swift` imports SwiftUI, behind `#if canImport(SwiftUI)`.
- `MarkdownColorScheme` keeps the core SwiftUI-free; the view maps `ColorScheme → MarkdownColorScheme`.
- Total mapping: any AST (incl. empty, deeply nested, malformed-but-parsed) maps without crashing. No force-unwraps on AST-derived strings.
- Reuse the engine's `ImageProvider` (re-exported) — do not redefine it.
